# ADR-001: Monorepo + Modular Monolith Foundation

- Status: Accepted
- Date: 2026-04-01

## Decision

Репозиторий строится как `pnpm` monorepo с `Turborepo`, а предметная логика организуется как modular monolith.

## Why

- команда из 5 человек быстрее двигается в одном репозитории;
- проще поддерживать единые boundaries, CI, tooling и onboarding;
- worker, web, portal и backoffice могут переиспользовать одни и те же contracts и modules;
- нет преждевременной сложности микросервисов.

## Structure

- `apps/*` — runtime applications
- `packages/modules` — business modules
- `packages/database` — Prisma schema и DB foundation
- `packages/jobs` — job contracts и queue foundation
- `packages/ui` — UI primitives
- `packages/config`, `auth`, `observability`, `testing` — platform packages

## Consequences

- масштабирование идёт по модулям и runtime-единицам, а не по отдельным репозиториям;
- границы поддерживаются через package API и import rules;
- shared-package не создаётся без отдельной необходимости.
