# AGENTS.md

## Scope

This file applies to `docs`.

## Purpose

The `docs` directory stores durable project knowledge.
It is for engineers, designers, operators, and leads.

## What belongs here

- architecture decisions
- ownership and review routing
- onboarding
- release and runbooks
- standards and import rules
- glossary
- product structure references

## Documentation rules

- Prefer concise, durable, practical documentation.
- Write for future teammates who were not in the original conversation.
- Keep docs aligned with actual repository structure.
- When architecture or process changes, update the related docs.
- Do not write aspirational fiction as if it already exists.

## Product-specific docs expectations

The product needs especially clear documentation for:

- role definitions
- moderation behavior
- anti-bypass logic
- content correction flow
- supplier request flow
- MVP boundaries

## ADR rules

- Record major decisions honestly.
- Include context, decision, consequences, and trade-offs.
- If a planned module is not yet implemented, say that clearly.
